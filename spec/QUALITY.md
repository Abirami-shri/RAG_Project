## 2. Git Workflow

### Branch naming
```
feature/<short-description>     e.g. feature/document-upload
fix/<short-description>         e.g. fix/sse-stream-close
chore/<short-description>       e.g. chore/update-deps
```

### Commit messages
Follow Conventional Commits:
```
<type>(<scope>): <subject>

feat(upload): add drag-and-drop file support
fix(chat): close SSE stream on client disconnect
chore(deps): bump openai to 1.35.0
docs(api): add /health endpoint docs
test(rag): add integration test for hybrid search
```

Types: `feat` `fix` `docs` `test` `chore` `refactor` `perf`

### PR checklist

Before opening a PR, verify every item:

```
Code
- [ ] All linting checks pass (ruff / eslint)
- [ ] All type checks pass (mypy / tsc)
- [ ] All formatting checks pass (black / prettier)
- [ ] New code has corresponding unit tests
- [ ] No hardcoded credentials, endpoints, or secrets
- [ ] No commented-out code left in

Tests
- [ ] Unit tests pass locally
- [ ] API tests pass locally
- [ ] Coverage did not drop below threshold

Behaviour
- [ ] Tested the golden path manually (upload → chat)
- [ ] Tested relevant error cases (bad file type, network failure)
- [ ] No regressions in existing features

Docs
- [ ] API changes reflected in Swagger/OpenAPI (auto-generated by FastAPI)
- [ ] Breaking changes called out explicitly in PR description
```

## 3. Observability

### 3.1 Logging

Backend uses Python's `logging` module with structured JSON output in production.

```python
# app/core/logging.py
import logging, json, sys

class JsonFormatter(logging.Formatter):
    def format(self, record):
        return json.dumps({
            "level": record.levelname,
            "logger": record.name,
            "message": record.getMessage(),
            "timestamp": self.formatTime(record),
            **({"exc_info": self.formatException(record.exc_info)} if record.exc_info else {}),
        })

def configure_logging(level: str = "INFO"):
    handler = logging.StreamHandler(sys.stdout)
    handler.setFormatter(JsonFormatter())
    logging.basicConfig(level=level, handlers=[handler])
```

**Log levels:**
- `DEBUG` — chunk boundaries, token counts
- `INFO` — document uploaded, processing started/finished, query received
- `WARNING` — fallback to PyPDF2 (Document Intelligence not configured)
- `ERROR` — processing failed, Azure service errors

**What to log at each stage:**
```python
log.info("upload.received", extra={"doc_id": doc_id, "size_bytes": size})
log.info("processing.started", extra={"doc_id": doc_id})
log.info("processing.completed", extra={"doc_id": doc_id, "chunks": n})
log.info("chat.query", extra={"conv_id": conv_id, "doc_scope": doc_ids})
log.info("chat.retrieved", extra={"conv_id": conv_id, "chunks": len(results)})
log.error("processing.failed", extra={"doc_id": doc_id, "error": str(e)})
```

### 3.3 Distributed Tracing

Each request gets a `correlation_id` injected at the router level and passed through all service calls. Logged with every line so log queries can reconstruct the full call chain.

```python
from fastapi import Request
import uuid

@app.middleware("http")
async def correlation_id_middleware(request: Request, call_next):
    correlation_id = request.headers.get("X-Correlation-ID", str(uuid.uuid4()))
    request.state.correlation_id = correlation_id
    response = await call_next(request)
    response.headers["X-Correlation-ID"] = correlation_id
    return response
```

## 5. Security Checklist

### Input validation
- [ ] File MIME type validated server-side (`content_type` from UploadFile)
- [ ] File size checked against `MAX_FILE_SIZE_MB` before processing
- [ ] Filenames sanitized — no path traversal (`../`), no special chars in blob key
- [ ] Chat message length capped (e.g., 4000 chars) to prevent prompt injection via input size
- [ ] `document_ids` filter validated — only UUIDs accepted

### Secrets
- [ ] `.env` is in `.gitignore`
- [ ] No secrets in source code or log output
- [ ] Azure credentials use minimum required permissions (Storage Blob Data Contributor, not Owner)
- [ ] AI Search uses admin key in dev, managed identity in prod

### Network
- [ ] CORS restricted to `CORS_ORIGINS` (not `*`)
- [ ] Blob containers are private (no anonymous read)
- [ ] Download links are signed URLs with 1-hour expiry

### Dependencies
- [ ] `pip-audit` run before each release: `pip install pip-audit && pip-audit`
- [ ] `npm audit` run before each release: `npm audit --audit-level=high`
- [ ] Dependency updates reviewed monthly (Dependabot PRs)

## 6. RAG Quality Metrics

Beyond technical correctness, the AI responses need to be evaluated for quality.

### 6.1 Faithfulness (no hallucination)

Every claim in the response should be traceable to a retrieved chunk. Spot-check by asking questions with known answers from a test document.

**Faithfulness test:**
```
Upload: a 3-page document about "Project Alpha launched in 2021 in Seattle"
Query: "Where was Project Alpha launched?"
Expected: "Seattle" with citation
Fail: any other city, or answer without citation
```

### 6.2 Relevance (retrieval quality)

The retrieved chunks should be about the topic asked, not random chunks from the document.

**Relevance test:**
```
Upload: a multi-topic document (history + science + economics)
Query: "What does the document say about economics?"
Expected: retrieved chunks are from the economics sections
Fail: economics query returns history chunks
```

### 6.3 No-context handling

When no relevant content exists, the model should refuse rather than hallucinate.

**No-context test:**
```
Upload: a document about cooking recipes
Query: "What is the GDP of France?"
Expected: "I couldn't find relevant information..."
Fail: any fabricated answer about GDP
```

### 6.4 Evaluation cadence

Run the above test suite manually against a curated fixture document (`tests/fixtures/eval_document.pdf`) before every release. Log pass/fail in the release notes.

## 8. Release Checklist

Before tagging a release:

```
Pre-release
- [ ] All tests green in CI
- [ ] Security audit clean (pip-audit + npm audit)
- [ ] Performance budgets met (Lighthouse + P95 latency check)
- [ ] RAG quality evaluation passed (faithfulness / relevance / no-context)
- [ ] DEPLOYMENT.md up to date

Release
- [ ] Tag follows semver: v<major>.<minor>.<patch>
- [ ] Release notes include: what changed, what was fixed, migration steps if any
- [ ] Docker images built and pushed to ACR with the release tag

Post-release
- [ ] Deployed to staging and smoke tested
- [ ] Key metrics baseline captured (first-token latency, processing time)
- [ ] Rollback procedure verified (revision list checked in Container Apps)
```
